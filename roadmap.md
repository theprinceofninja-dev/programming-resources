# Programming Roadmap

## Prerequisites
- Internet, Electricity, and Laptop
- Google everything — solutions are similar across problems

## Mindset
- Problem solving is the goal; code is just the tool
- CS50 is a must
- Learn by time and patience

## Language & Fundamentals
- One language deeply — start with Python
- Data structures, syntax, control flow, loops
- Learn scripting to automate manual tasks
- Databases are a good entry point

## Projects
- Personal projects and hackathons build real experience
- Good starter project: Stateful counter / CRUD app
- Full stack: learn JavaScript (React, Express/Node, TypeScript)
- Hosting and cloud providers

## Essential Backend & Dev Skills
- Bash / Linux
- Git / GitHub (branches, pull, commit, cherry-pick)
- HTTP / Networking basics
- Databases (SQL + ORM)
- APIs (REST)
- Docker / Kubernetes
- Testing (unit testing, integration testing — write testable code)
- System Design (database schema, classes, functions, UML)
- Scalability (algorithms, data structures, database performance)
- Documentation (from requirements to code)

## Real-World Junior Developer Onboarding Plan
Taken from an actual workplace training plan — skills expected from new junior developers:

**Core technical:**
- Python
- Bash & Linux in general
- MySQL (databases)
- JSON
- Git, gitk, gitlab
- Docker
- Awk scripting
- Redis
- Regular expressions (REGEXP)
- HTTP & basic web

**Dev tools & workflow:**
- VSCode
- Agile / Scrum
- Standups format: Who are you, What did you do, What are you working on, Any blockers?
- Jira (task tracking), Confluence (documentation), Wireshark (network inspection)

**Practice scenario:**
1. Learn a programming language
2. Learn databases
3. Generate reports using the database
4. Build a web interface that talks to a backend to query the database

## Interview Prep
- Strings, arrays, DFS, recursion

## Dev Environment
- Terminal: Linux or Bash for Windows (ls, mv, mkdir...)
- Editor: VSCode (a must), IntelliJ
- Version control: Git
- Cheat sheets

## Key Concepts (Fundamentals)
- Assignment: storing data in variables (type, name, value, memory address)
- Sequential execution — with conditionals and jumps
- Loops: conditional repetition
- Encapsulation: grouping instructions into functions/subroutines
- Arithmetic operations, input/output
- Compound data structures (arrays, lists, maps...)
- Every problem has an algorithm
- Tests and conditions
